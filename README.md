# 📋 UrFU Practice Docs Automation

> Десктопное приложение для автоматического заполнения документов по учебной практике. Загружаете Excel со списком студентов и шаблоны Word — программа сама подставит данные в нужные поля и сохранит готовые `.docx` (опционально с конвертацией в `.pdf`).

Создавалось под отчётность по учебной практике УрФУ, но подойдёт любому преподавателю, кому нужно массово заполнять однотипные документы по табличным данным.

---

## 🚀 Что умеет

- **Массовое заполнение шаблонов Word** — одной кнопкой формирует комплект документов для каждого студента из списка (по умолчанию 4 типа документа: индивидуальное задание, извещение, отзыв, отчёт).
- **Сохраняет исходное форматирование** — шрифты, размеры, жирный, курсив, подчёркивание не «слетают» даже если плейсхолдер разбит на несколько фрагментов внутри документа.
- **Автоматически определяет столбцы Excel** по заголовкам (с учётом кириллицы и распространённых вариаций написания).
- **🆕 Пред-валидация шаблонов** — перед обработкой проверяет, что в каждом `.docx` действительно есть плейсхолдеры `{{ ... }}`. Если выбрали не тот файл — программа сразу предупредит, а не сделает «пустышки».
- **🆕 Строгая проверка дат с понятными ошибками** — если в Excel вместо даты текст («неизвестно», «—» и т.п.), в логе появится сообщение с **именем поля** и **номером строки**, чтобы было ясно, где именно ошибка.
- **🆕 Устойчивость к плохим данным** — если у одного студента кривые данные, он пропускается, а обработка остальных **продолжается без остановки**.
- **Умное форматирование данных:**
  - ФИО руководителей сокращается до формы `Иванов И.И.`
  - Даты приводятся к виду `DD.MM.YYYY`
  - Дата согласования — `«DD» месяц YYYY` с русским названием месяца
  - Курс вычисляется из номера группы автоматически
  - Сроки этапов парсятся из формата `DD.MM.YYYY-DD.MM.YYYY`
- **Конвертация в PDF** (опционально, через Microsoft Word) — `Только DOCX` или `DOCX + PDF`.
- **Графический интерфейс** на Tkinter: выбор файлов мышкой, прогресс-бар, лог выполнения в реальном времени, всплывающее уведомление по окончании.
- **Многопоточность** — UI не зависает во время обработки.

## 📦 Установка

### Вариант 1: запустить готовый `.exe` (Windows)

Скачайте `ultimo.exe` из релизов и запустите. Никакого Python устанавливать не нужно.

> ⚠️ Для конвертации в PDF на компьютере должен быть установлен Microsoft Word.

### Вариант 2: запустить из исходников

```bash
git clone https://github.com/<ваш-username>/<имя-репо>.git
cd <имя-репо>
pip install -r requirements.txt
python ultimo.py
```

**Зависимости:**

```
openpyxl
python-docx
pywin32   # только для PDF-конвертации, только Windows
```

## 🛠 Использование

1. **Подготовьте Excel** со списком студентов. Первая строка — заголовки, ниже — данные. Поддерживаемые столбцы (порядок не важен):

   | Столбец | Назначение |
   |---|---|
   | ФИО | ФИО студента |
   | Академ. группа | Номер группы (например, `УГИМ-240030`) |
   | Направление | Код/название направления |
   | Начало практики / Окончание практики | Даты |
   | Начало срока сдачи отчёта / Окончание срока сдачи отчёта | Даты |
   | Руководитель практики от организации | Полное ФИО |
   | Руководитель практики от предприятия | Полное ФИО |
   | Должность руководителя от предприятия | Текст |
   | Наименование предприятия | Текст |
   | РОП магистратуры | Полное ФИО |
   | Срок организационного / основного / заключительного этапа | `DD.MM.YYYY-DD.MM.YYYY` |
   | Согласовано предприятие | Дата |
   | Дата приказа / Номер приказа | Дата и текст |

   > 💡 **Важно про даты:** в ячейках с датами должен быть либо настоящий Excel-формат даты, либо строка вида `DD.MM.YYYY`. Если там будет произвольный текст, программа сообщит об ошибке и пропустит этого студента.

2. **Подготовьте шаблоны Word.** В исходных документах замените реальные данные на плейсхолдеры в фигурных скобках, например:

   ```
   Студент: {{ ФИО }}
   Группа: {{ ГРУППА }}, {{ КУРС }} курс
   Направление: {{ НАПРАВЛЕНИЕ }}
   Предприятие: {{ ПРЕДПРИЯТИЕ }}
   Руководитель от предприятия: {{ РУК_ПРЕД }}, {{ ДОЛЖНОСТЬ }}
   Сроки: с {{ НАЧАЛО }} по {{ ОКОНЧАНИЕ }}
   Согласовано «{{ ДАТА_СОГЛАСОВАНИЯ }}»
   ```

3. **Запустите программу** и в окне:
   - Укажите Excel-файл со студентами.
   - Выберите 4 шаблона Word (или меньше, если нужен не весь комплект).
   - Выберите папку для сохранения результатов.
   - Выберите формат: `Только DOCX` или `DOCX + PDF`.
   - Нажмите **«Сформировать документы»**.

4. **Готово.** В выбранной папке появятся файлы вида:

   ```
   Иванов_ИИ_УГИМ-240030_ИндЗадание.docx
   Иванов_ИИ_УГИМ-240030_Извещение.docx
   Иванов_ИИ_УГИМ-240030_Отзыв.docx
   Иванов_ИИ_УГИМ-240030_Отчет.docx
   ```

## 🛡 Обработка ошибок

Программа умеет распознавать и аккуратно сообщать о типичных проблемах:

| Что случилось | Что покажет программа |
|---|---|
| Выбран не тот файл шаблона (нет плейсхолдеров) | Прервёт работу до начала обработки и укажет, какой именно файл «не тот» |
| В дате текст вместо даты (например, «уточнить») | Пропустит студента, в логе укажет имя поля и номер строки в Excel |
| Пустое ФИО в строке | Пропустит строку с пометкой в логе |
| Шаблон не открывается / повреждён | Сообщит, какой файл сбойнул |
| Один студент сломался | Продолжит обработку остальных |

Все ошибки видны в реальном времени в логе внутри окна программы.

## 🏗 Сборка в `.exe`

Сборка выполняется через PyInstaller. Скрытые импорты нужны для работы PDF-конвертации в собранном виде:

```bash
pip install pyinstaller
pyinstaller --noconfirm --onefile --windowed ^
  --hidden-import win32com ^
  --hidden-import win32com.client ^
  --hidden-import pythoncom ^
  --name ultimo ^
  ultimo.py
```

Готовый файл появится в папке `dist/`.

## 🧱 Архитектура

Один файл `ultimo.py` для простоты распространения. Логические блоки:

- **Форматирование** — функции `fmt_date`, `fmt_date_ru`, `shorten_name`, `get_kurs`, `parse_stage_dates`. Кастомные исключения `DateError` и `TemplateError` для понятной диагностики.
- **Замена в Word** — `_replace_in_para` корректно обрабатывает случай, когда плейсхолдер разбит на несколько `run`'ов внутри параграфа (типичная проблема `python-docx`); `fill_template` обходит параграфы, таблицы, колонтитулы.
- **Валидация шаблонов** — `validate_template` проверяет наличие хотя бы одного плейсхолдера `{{ ... }}` перед запуском полной обработки.
- **Чтение Excel** — `read_excel` + словарь `EXCEL_COLS` для распознавания заголовков по алиасам. Каждой строке присваивается номер для адресной диагностики ошибок.
- **Построение замен** — `build_replacements` собирает словарь `плейсхолдер → значение` для одного студента, с проверкой корректности дат.
- **Генерация** — `generate_documents` сначала проверяет все шаблоны (Шаг 0), потом создаёт DOCX (Шаг 1), при необходимости запускает Word через `pywin32` для пакетной конвертации в PDF (Шаг 2). Если у одного студента ошибка — пропускает его и продолжает.
- **GUI** — класс `App` на `tkinter`, обработка идёт в отдельном потоке, прогресс и лог обновляются через `after`.

## ⚠️ Ограничения

- PDF-конвертация работает **только под Windows** и требует установленного Microsoft Word (используется COM-автоматизация через `pywin32`). Без Word можно работать в режиме «Только DOCX» на любой ОС.
- Программа ожидает заголовки столбцов в первой строке Excel.
- Плейсхолдеры строго в формате `{{ ИМЯ }}` (с пробелами внутри фигурных скобок).
- Даты в Excel должны быть либо настоящим Excel-форматом даты, либо строкой `DD.MM.YYYY` / `DD/MM/YYYY` / `DD-MM-YYYY`.
